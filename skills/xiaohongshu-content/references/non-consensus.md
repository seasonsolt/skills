# 非共识选题

小红书上同一话题的笔记多如牛毛，**写共识 = 被淹没**。"两个都好各有所长""结论前置"这类全网写烂的角度，读者一眼扫过。真正拉人停留和互动的，是**没人愿意写的那句实话**。

## 方法：全网检索真实用户声音

不要凭记忆写。去检索真实用户的**争议、吐槽、反直觉数据**：

- **中文**：V2EX、知乎、少数派、linux.do
- **英文**：Reddit（对应 subreddit）、Hacker News、行业 Medium
- **权威/学术**（关键，见下节）：Google Scholar / arXiv 的同行评审研究、知名专家的博客/Substack（如 Ethan Mollick 的 oneusefulthing）、名家在访谈/论文里的原话

### 混合检索路由

不要让一个搜索源包办所有查询。先按问题类型选主检索器，再用另一条链路核验关键事实：

| 查询类型 | 主检索器 | 何时切换 / 补查 |
|---|---|---|
| 中文社区、用户吐槽、产品价格、官方技术文档 | **Tavily**（可用时） | 结果少于 3 个独立来源、没有一手来源或正文片段不足时，补 Codex/WebSearch |
| 论文、benchmark、研究限制、正反证据 | **Codex/WebSearch** | 找到结论后，用 Tavily 补中文讨论和用户体感 |
| 最新发布、价格、版本、规则等时效事实 | **两路都查** | 按官方域名和真实发布日期重排；网页更新时间不能冒充事件日期 |
| 小红书站内内容、登录后数据 | **浏览器/站内搜索** | Tavily 和公开 WebSearch 都不能替代登录态 |

执行规则：

1. 每个关键数字至少有 **2 个独立来源**，其中至少 1 个是一手来源（官方文档、论文原文或当事人原帖）。
2. 两路结论冲突时不投票：回到原始页面核对发布日期、实验条件和原话上下文；无法消解就把分歧写进选题依据。
3. Tavily 不可用、超时或结果不够时，自动回退 Codex/WebSearch；`WebSearch` 不可用时，再用浏览器打开搜索引擎。
4. 检索返回的 AI 摘要只能用来定位页面，不能作为数字或引文的最终出处。

检索技巧：
- Tavily 优先取搜索结果的网页片段/正文，不直接采信它生成的 `answer`。
- `WebSearch` 工具**可能不可用/报 unavailable**——直接用浏览器 MCP 打开 Google 搜（`get_page_text` 抓结果页）。
- Reddit 用 `WebFetch` 常被拒 → 用浏览器 MCP 直接打开帖子读。
- 搜 "X vs Y honest / reddit / 封号 / 性价比 / switched" 这类词，能挖到真实体感而非软文。

## 提炼非共识点

好的非共识点长这样（每条都要能落地）：
- **有具体出处**：引 V2EX 原话、Reddit 高赞帖、知乎某答主的原观点。
- **有数字**：具体价格、具体倍数（"额度远大于""3-4x token""400 刀砍到 100 刀"）、具体对比。
- **反直觉**："海外重度用户站 A，但国内博主都在教 B"这种撕裂感。
- **本地化视角**：国内特有变量（如封号率、支付、网络）往往是海外测评忽略、但国内用户最关心的第一决策因素。
- **留接话空间**：非共识负责让人点进来/收藏，但角度可以新颖到"无话可说"。要评论，还得在角度里留个读者能接的坑（站队 / 可反对 / 可 +1 / 可补反例）——见 [`接话空间.md`](接话空间.md)。

## 反共识必须挂权威（否则就是哗众取宠）

反共识观点如果只是"我觉得"，就是博眼球、站不住；**挂上权威背书，才从"哗众取宠"变成"内部人带你看真相"**——这也是"真从业者"人设的可信度根基。每个反共识主线，尽量配至少一条：

- **同行评审研究 + 硬数字**（最强）：如"MIT 54 人脑电图实验发现用 ChatGPT 组大脑连接最弱""666 人研究：越依赖 AI 批判性思维越差"。
- **知名专家的原话/论调**：给个台阶也给权威，如沃顿 Ethan Mollick"AI 用不好才会让你变笨，而用不好是默认"。
- **国内外论坛热帖**：登上 HN/知乎热榜的讨论，代表"很多人有同感"。

组合技最好：**研究给震惊 + 专家给定调 + 热帖给共鸣**，你以"看透真相的从业者"身份收口。
⚠️ 研究要标清出处，别夸大成"铁证"（样本小/有争议就说"研究发现/有点吓人"）；专家的话别断章取义。

## 受众权衡：够反共识，但别太窄

反共识 ≠ 越硬核越好。**选题太技术/太深 = 受众太小，传不开**（如"多 agent 怎么 handoff""skill 内部结构"——只有一小撮开发者看）。

- 保留你的**护城河**（从业者身份、说实话），但把**对象换成大众场景**：学习/自我提升、职场提效、副业变现、认知避坑。
- 判断法：这题除了同行，**普通打工人/学生/家长能不能一句话看懂并关心**？不能就太窄。
- 同一个专业洞见，往上抽象一层就能破圈：把"AI 写代码让我失控"抽象成"过度依赖 AI 让你把该自己长的本事外包了"，人人有感。

## 制造互动的暗招

- **暗讽同行**（"国内自媒体还在教大家用 X"）——评论区容易吵，吵架就是互动。
- **损失厌恶**（"很多人第一个月就多花了冤枉钱"）——比"帮你省钱"更戳。
- **对号入座**（"你是不是也这样👇"）——让读者觉得在说自己。

## 红线

- 数字/引用发布前**核实**，尤其价格、版本（模型迭代快，两个月前的帖子可能过时）。被评论区指出错误会伤"亲测"人设。
- 观点可以尖锐，但**别造谣、别编数据**。非共识 ≠ 编故事。
