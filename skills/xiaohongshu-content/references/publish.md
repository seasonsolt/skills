# 发布

## 上传图片（硬坑，务必如实告诉用户，别硬来）

- 若用户显式要求 Chrome / 浏览器 / 现有登录态，先走对应浏览器能力。浏览器不可用时，报告具体阻塞点（扩展未安装/未启用、profile 不对、未登录、上传控件不可用），不要静默换后端。
- 浏览器 MCP 的 `file_upload` 有**安全白名单**：只接受"用户主动分享给会话的附件"，**不接受任意本地路径**。实测下载文件夹、项目目录、用户自建的 `uploads` 文件夹**全被拒**。
- ChatGPT 和小红书是**跨站**，浏览器里没法直接把图从一个站搬到另一个站。
- 🚫 **绝不**把图片文件塞进扩展的内部会话目录去骗过校验——那是**绕过安全控制**，会被 auto-mode 分类器拦下，也不该做。
- 结论：**多数环境下自动上传图片走不通**，最后一步交给用户手动，这是工具的硬限制，不是操作失误。诚实说明 + 把用户的手动量压到最低。

给用户的手动收尾（约 30 秒）：
1. 草稿编辑器保持打开（或从「草稿箱」重新打开草稿）。
2. 访达打开存图的文件夹（本项目 `posts/<...>/cards/`），**框选全部卡片图**（按 P1→Pn 排序），拖进"图片编辑"区。
3. **删掉占位封面**（悬停第一张占位图出现删除按钮）。
4. 核对：第一张是封面、最后一张是收尾，顺序没乱。
5. 点**发布**。

（唯一合规的自动路径：若用户**真正把图作为附件分享进会话**，`file_upload` 才可能接受，再传到图片输入框并删占位。）

**完成判据**：全部卡片按 P1→Pn 在草稿内、占位已删、顺序已核对，笔记已发布。
