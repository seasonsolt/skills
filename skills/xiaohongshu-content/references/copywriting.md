# 结构化文案

一篇笔记 = **封面图 + 若干图片 + 一段正文文字 + 标签**。正文不是图片，是发布时打字区里的纯文字，它的前两行会出现在信息流摘要里，决定别人点不点进来；里面的关键词和标签是给算法看的。

## 情绪价值前置（决定能不能传播）

**回报必须给在"读的当下"，不能锁在"实践之后"。** 读者划完最后一张卡，就该已经收获了情绪——震惊、被戳中、解气、松一口气、涨谈资、一点优越感。若价值是"你照着做才有用"，那是延迟满足：只换收藏（"以后再说"），换不来转发。而互联网传播＝转发，转发＝读时就爆发的情绪。

- **观点/情绪/认知贴**（即时情绪）→ 传播/转发；**教程/清单贴**（延迟价值）→ 收藏/留存。要流量先做前者。
- 就算内容里有方法，也要把**情绪和结论前置**、方法降为次要，甚至丢给"下一篇/评论区"，让读者读这一篇就情绪饱满。
- 自检：每张卡念一遍，问"读者读完这张，此刻什么感受？" 若答案是"得去试试才知道"，这张就没在给情绪。

## 正文骨架

```
[钩子开头]   前 1-2 行，命中目标读者痛点 + 给一个停留理由（亲测/10分钟/一次配好）
[结论前置]   先给答案/结论，别让读者猜
[分步骤]     ①②③④ 清晰编号，一步一行
[系列 + CTA] 系列定位 + 下期预告 + 评论引导 + 关注引导 + 回引旧笔记
[出处]       转载内容标明来源
```

## 排版与可读性（小红书原生）

正文是"笔记"不是"文章"：成段的书面语转化差，要拆成**短段落 + emoji 分段标记**，让人划得动、扫得清。

- **短段落**：一段 1–3 行，一个意思一段，段间留空行。
- **emoji 当小标题**：关键节点用 emoji 起头分段（🔥事件 / 👇起因 / ⚠️避坑 / 📊数据 / 🧊反转 / 💡观点 / 📮预告），并列项用「·」列点。
- **别滥用**：emoji 是锚点不是装饰，每段至多一个起头，别每行都挂。

⚠️ **与去 AI 味的取舍**：humanizer-zh 第 17 项把 emoji 当 AI 痕迹要求删——那是针对**机械堆砌的装饰性 emoji**。小红书这种**有意的分段锚点 emoji 属于平台原生排版，要保留**。做法：prose 严格过 humanizer 并打分 ≥40/50（分段 emoji 不计入 AI 痕迹扣分），排版层的 emoji 照留。

## 四个转化机制（结尾必备）

这是把"看过"变成"关注"的核心，按这四条写结尾：

1. **系列化**——"这是【XX 系列】第 N 篇"。把单篇变成"追更对象"，是收藏党转关注的最大杠杆。
2. **下期预告**——具体到选题（"下一篇：A 和 B 怎么选，很多人买贵了"），用损失厌恶勾人。
3. **评论引导 → 设计接话空间**——光结尾问一句还不够：问题太难答（要专业才接得上），读者照样收藏就走。要把"读者非说一句不可的坑"设计进**内容本身**——站队题 / 可 +1 的共同吐槽 / 可反对的具体断言 / 窄到读者有反例；CTA 落在这个坑上，别泛泛"你怎么看"。每个接话点过一句门槛护栏"非本领域的人能一句话接上吗"。方法见 [`接话空间.md`](接话空间.md)。回评能二次推流。
4. **收藏拦截**——在收藏动作前塞关注请求（"收藏前顺手点个关注"）。若该账号是收藏型受众，这条最有效。

**系列联动**：回引主页的旧笔记（"上一篇：… 主页可看"）。两篇互相导流，还能盘活沉掉的老笔记。若旧笔记只"提出问题"，新笔记正好"给解药"，联动效果最好。

## 规则与坑

- **标题 ≤ 20 字**（含标点）。要有钩子，别平铺。
- **数字先核实**（价格、版本等），别被评论区打脸。
- **"我的用法"必须真实**——这一节是全文最有说服力的地方，编的会露馅。
- **改已发布笔记 = 重新审核**，有限流风险，尤其正文含外链/礼品卡。要改就**增量修改**（只在头尾加钩子，别动主体、外链、标签），把重审风险降到最低。
- 承诺了"下一篇讲 X"就要兑现，别让追更的人落空。

## 去 AI 味（读起来像真人，不像 AI）

AI 写的文案有种"理性优越感"，一眼能认出、也最劝退。病征：
- 工整的一二三四教科书结构；堆术语当权威（越堆越像论文）；
- "说句得罪人的实话""真相是"这种**居高临下、我来点醒你**的腔调；
- 理性周全但没人味，没有犹豫和情绪。

解法——写成**真人跟朋友唠嗑**：
- 第一人称，有**真实的犹豫和自嘲**（"这不就是我吗""说实话我又舍不得""我也踩过"）；把"我懂真相"换成"我也慌，跟你一样"。
- 口语、短句、自然流动，别硬套一二三四；术语用**大白话**当场翻译，别拿它撑权威。
- 但**别矫枉过正**：去 AI 味 ≠ 越癫越好，硬凹网络烂梗、堆语气词、夸张失真会毁可信度（尤其科普/权威贴）。

**流程（🔴 强制，非可选）：正文写完，必须调用 `humanizer-zh` 过一道去 AI 味——默认执行、不需用户点名、未过不算完成。**
- **调用 humanizer-zh**（Claude Code：`/humanizer-zh [正文]`；Codex/其它无斜杠命令的 agent：读并套用 [`.claude/skills/humanizer-zh/SKILL.md`](../../humanizer-zh/SKILL.md) 的方法）——已随本仓库 `.claude/skills/`，新会话自带：按 24 项 AI 痕迹改写 + 50 分 rubric 打分，**≥40/50 才算过**，不到就继续改，并向用户报一句改了什么 + 分数。工具全表 / 即用提示词 / 检测器见 [`去AI味-工具与提示词.md`](去AI味-工具与提示词.md)。
- **仅当 humanizer-zh 不可用**（未安装/报错）才 fallback：过一道 DeepSeek 母语重写。key 在 `.env`（`DEEPSEEK_API_KEY`），模型 `deepseek-chat`，**temperature=1.0**（实测 1.3 会玩脱成"发疯文学"、语无伦次）。
- ⚠️ 去味时**事实、数字、出处一个字不能改**；模型（含 DeepSeek）常**替你编造第一人称经历**——必须换成你真实的事（红线：亲测/我的用法必须真）。

## 增量优化已发布爆款的模板

若要给一篇已经在涨的笔记加关注钩子（而不重写）：
- **开头加一行钩子**（进信息流摘要、命中痛点）
- **结尾加"关注转化块"**（上面四个机制）
- 中间主体、外链、标签**一律不动**
