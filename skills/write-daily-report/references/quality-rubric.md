# 日报质量量表

将图中可辨认的质量分析维度整理为五项检查。每项按 0–20 分内部自评，总分 100；评分只用于改写，不在默认输出中展示。

## 1. 内容完整性（20）

- 18–20：重要任务、状态、成果/产出、问题和明日安排齐全，关键事实无明显缺口。
- 12–17：主体事项齐全，但少量结果、状态或后续安排不明确。
- 0–11：只列动作或遗漏重要事项，读者无法判断完成情况。

## 2. 结构化表达（20）

- 18–20：按项目/目标归类，层级清晰，一条事项形成完整逻辑链，重点突出。
- 12–17：基本可读，但仍有重复、顺序混乱或长句堆叠。
- 0–11：按时间机械罗列、碎片化严重，依赖读者自行拼接信息。

## 3. 目标与结果导向（20）

- 18–20：说明目标或业务背景，并给出数字、交付物、里程碑、结论或影响等可验证结果。
- 12–17：有明确产出或进度，但目标、价值或完成标准不够清楚。
- 0–11：主要是“沟通、开会、跟进、处理”等动作，无法判断产出和价值。

## 4. 问题及改善（20）

- 18–20：明确问题、影响、原因（仅在已知时）、已采取动作、责任边界和下一步，形成闭环。
- 12–17：能识别问题并提出处理动作，但影响或后续不够具体。
- 0–11：回避问题，或只描述困难、抱怨和外部责任，没有改善方案。

## 5. 明日计划（20）

- 18–20：按优先级列出可执行动作，包含对象、预期产出、完成标准或时间节点。
- 12–17：有明确事项，但优先级、产出或验收标准不完整。
- 0–11：无计划，或仅写“继续跟进/推进/处理”等不可验收表述。

## 改写门槛

默认以总分不低于 85 为目标，并满足以下硬约束：

- 不得为了提高分数而补造事实。
- “结果导向”不等于必须有数字；明确交付物、结论、里程碑和风险消除同样是结果。
- 若用户当天确实没有问题、亮点或量化数据，如实省略，不用套话填充。
- 待外部反馈不等于未推进；写清已完成动作、等待对象、影响和下一次跟进节点即可。

## 常见低质量表达与改写方向

| 低质量表达 | 改写方向 |
|---|---|
| 开会讨论项目 | 与谁围绕什么问题达成何种结论，仍有哪些待办 |
| 跟进需求 | 跟进对象、当前状态、已推动结果、下一节点 |
| 完成相关工作 | 具体交付物、数量/范围、完成状态和影响 |
| 项目有问题 | 问题、影响、已采取措施、下一步和所需支持 |
| 明天继续推进 | 推进哪一步、预期产出、完成标准或截止时间 |
